# 01 · 建立 Codebook

> **Neuman 出处**：Ch 12, "Coding the Data" + "Codebook" · p.393-394
>
> **何时用**：拿到原始数据 (问卷、记录表、或公开数据集), 还没整理成可分析格式时
>
> **前置**：原始数据已收集 / 已下载

---

## 📖 Neuman 怎么讲

> **Codebook**: A document that describes the procedure for coding variables and their location in a format that computers can use.
>
> 编码本必须在收集数据**之前**或**刚开始时**就建立。包括：
> - 每个变量的名称、含义、取值范围
> - 缺失值的代码（如 -1, -2, blank）
> - 该变量在数据文件里的列位置
> - 度量层级（nominal / ordinal / interval / ratio）

**关键**：Codebook 一旦丢了, 数据也就废了。

---

## ⚠️ 必须你自己判断的部分

- **变量的概念定义**：什么是"知识工作者"？什么是"满意度"？这是理论判断, AI 给不了
- **度量层级**：把"年龄"当连续变量还是分组？是 ratio 还是 ordinal？
- **缺失值的处理立场**：是删除缺失？还是当作单独类别？

---

## ✂️ Prompt 模板

```
我要为以下数据建立 codebook (Neuman 的格式)。

【数据来源】[填: 我自己做的问卷 / 公开数据集 X / 访谈转录数据 / ...]

【数据形态】[填: CSV 文件 / Excel / 文本逐字稿 / 还没数字化的纸质问卷]

【已有变量列表】（如有）
[贴变量名 + 含义, 一行一个; 如还没整理就写"未整理, 我描述给你看"]

【关键概念定义】（必填——这是研究者的判断）
- 我的核心 IV (自变量) 概念定义: [填一两句话]
- 我的核心 DV (因变量) 概念定义: [填一两句话]
- 我的控制变量 (列出): [可以列 3-5 个]

【请帮我做】
1. 列出每个变量, 标注:
   - 变量名 (英文短码, ≤10 字符)
   - 含义 (中文一句话)
   - 度量层级 (nominal/ordinal/interval/ratio) — **如果不确定**, 给我两个选择 + 各自的代价
   - 取值范围
   - 缺失值代码
   - 在数据文件里的列位置 (如已知)
2. 标记**理论关键变量**（IV/DV/Mediator/Moderator/控制）
3. 输出格式: Markdown 表格 + 一段附注说明

【硬约束】
- 不要假设我的变量含义, 不确定就反问我
- 度量层级有歧义时, 列出选择让我决定, 不要替我决定
- 不要发明数据里没有的变量
```

## 💡 执行后的下一步

- 如果数据已录入 → 用 [02 · 清洗数据](02-clean-data.md) 做 wild code 和 contingency 检查
- 如果数据还没录入 → 拿这份 codebook 去录数据 (Neuman p.395-396 讲了 4 种录入方式)
- 如果发现概念定义有问题 → 回到 第4讲 重新做研究设计, 不要硬上

## 📌 WVS 演示填法

```
【数据来源】WVS Wave 7 China 2018 公开数据集 (n=3036)
【数据形态】CSV 文件, 分号分隔, 346 列
【已有变量列表】Q158-Q163 (科技态度), Q260 (出生年), Q275 (教育), Q281 (职业), Q287 (收入十分位), H_SETTLEMENT (城乡)
【关键概念定义】
- IV: 知识工作者 = Q281 ∈ {1, 2}（专业技术 / 高级管理）
- DV: 把科技视为机会 = Q159 (10-pt Likert)
- 控制: 年龄 (从 Q260 算)、教育 (Q275)、收入 (Q287)、城乡 (H_SETTLEMENT)
```
