# Model Paper Deconstruct · 动静虚实四产出（高密度版）

对一篇 Model Paper 做完整拆解，产出 4 个动静虚实产出。

## 调用约定（独立运行）

**必填（二选一）**：
- PDF 路径 → 直接读取该 PDF 全文后再拆解；
- 论文标题 → 调用 Zotero MCP（zotero_search_items / zotero_get_item_fulltext / zotero_semantic_search）在用户 Zotero 文库中检索并提取全文。**若 Zotero 中找不到，明确告知并停止，不要使用网络资料替代。**

文献信息（Author Year · Title · Journal · DOI）：用户给了就用；没给就从论文中提取后补全。

**输出落盘**：`Model-Paper-拆解-[AuthorYear].md`（默认当前工作目录；[AuthorYear] 替换为实际作者+年份，如 Manolev2019）。

**产出语言**：项目 CLAUDE.md 或跟随用户。

- **调用方覆盖约定**：调用方（工作台 prompt / 用户）显式给出的参数、输入路径、落盘路径与工具选择，一律覆盖本 skill 的默认；但本 skill 的方法步骤、硬约束与停点不可被省略或稀释——调用方若要求跳过某硬约束，以本 skill 为准并提示冲突。

## 【动静虚实框架定义】（拆解基础，必须理解后再拆）

动静虚实是一个 2×2 分析框架，用于解析社科论文在做什么：

```
        │  静（状态·State）   │  动（过程·Process）
────────┼───────────────────┼──────────────────────
  实    │   静实              │   动实
（可观察）│ 可观察的存在状态     │ 可观察的变化过程
────────┼───────────────────┼──────────────────────
  虚    │   静虚              │   动虚
（概念化）│ 命名/分类这个对象    │ 解释这个变化的机制
```

- 横轴 静/动：研究的是"状态"（可拍照）还是"过程"（需录像）？RQ 含"X 是什么/如何分布" → 偏静；"X 如何变成 Y / Y 如何发生" → 偏动。大多数社科论文是静 + 动混合。
- 纵轴 实/虚：处理的是可观察现象（实）还是概念化后的抽象（虚）？感官可直接触及 → 实（访谈引语、样本数据、设备、政策文本）；需学术训练才能"看见" → 虚（理论术语、分析范畴、命名）。注：实/虚 ≠ 量化/质性，质性研究也有静实（访谈引语本身就是静实）。
- 各象限典型内容：
  - 静实：研究对象的可观察状态（参与者信息、样本描述、数据来源、时间地点等）
  - 动实：研究对象的变化过程（具体行为序列、互动过程、访谈中实际发生了什么）
  - 静虚：命名这个对象的核心概念（理论术语、分析范畴、关键学术词汇）
  - 动虚：解释变化的机制——⭐ 最关键。必须是因果生成机制，不是装饰性"理论框架"。区分：动虚 = mechanism（机制）≠ theoretical framework（框架名称）。例：Foucault 的 subjectivation / Sassen 的 expulsions / Bourdieu 的 capital conversion。误区："我用了 Foucault 框架" 不等于动虚——要问：这个框架的哪个机制解释了你观察到的哪个变化过程？
- 象限间的 3 个移动（= 论文元结构）：
  1. Conceptualisation（实→虚）Intro + Lit Review — 从现象到命名
  2. Operationalisation（虚→实）Methods — 把概念翻译成研究操作
  3. Re-conceptualisation（实→虚）Discussion — 从数据扩展/修订原有理论
- 任何经验论文都在做这 3 个移动，4 格都必须填满。纯理论文章只做移动 1；量化论文的移动 2 表现为"操作化定义 + 测量"。

## 【信息量与密度要求：必须执行】

不要写成摘要版。目标是"可直接作为论文结构模仿材料"的高密度拆解。

总长度要求：Markdown 正文不少于 6000 中文字。不允许只给概括性判断；每个判断必须落到"论文中的具体节次 / 具体概念 / 具体案例 / 具体引用角色"。宁可多写机制链条，也不要泛泛复述论文主题。

**产出 1 · 四象限填充表**：
- 每格至少 5 条，动实可写 5-7 条。
- 静实必须包含样本、时间、地点、数据形式、使用场景、关键可观察事实。
- 动实必须包含具体行为过程或访谈案例，例如"上传自传""询问政治选择""用 AI 改邮件"等。
- 静虚不能只列概念名，必须写出每个概念在本文中的定义或作用。
- 动虚是最重要部分，每条必须写成"机制链"：A → B → C → D，并说明该机制解释了论文中的哪个经验变化。动虚至少 5 个机制，每个机制不少于 80 字。

**产出 2 · 虚实转换三过程轨迹**：
- 不要每个移动只写一句话。每个移动至少拆到具体小节（如 Sec 1、Sec 2.1、Sec 2.2、Sec 3.4、Sec 5.1）。
- 每个小节说明它完成了什么转换："从什么实证现象出发 → 用什么概念命名 → 如何服务论文主论证"。
- Re-conceptualisation 必须逐一说明 Discussion 各子节如何把 Results 中的发现重新理论化。

**产出 3 · 三种引用角色清单**：
- 每条引用三列：文献、出现位置、功能。
- 锚定引用不能只列文献名，要说明它如何"开场定调 / 背景铺垫 / 方法背书 / 限制说明"。
- 定位引用必须说明它勾勒了哪条领域脉络，以及本文如何借它定位研究缺口。
- 框架引用必须说明：① 出现路径（Intro → Theory → Methods → Discussion → Conclusion）② 提供的核心机制 ③ Discussion 中如何被回收。
- 最后计算两种比例：① 框架引用数 / 全文参考文献总数 ② (框架 + 定位) / 全文参考文献总数。

**产出 4 · Section → 象限功能映射**：
- 按论文真实小节列出，不要合并 Results 或 Discussion。
- 每节的"节功能"不能只写摘要，必须说明该节在论文结构中的动作（如"命名研究缺口""构建机制""操作化理论""延迟理论解释""回收结果并扩展概念"）。
- 引用数必须实测；估算必须标"约"。对引用为 0 的节要特别说明其结构意义。

**末尾追加**：
- 反直觉发现（≥3 条，每条 ≥120 字）：必须解释"为什么反直觉"以及"这个结构安排有什么写作功能"；不能只说现象（如"Results 引用为 0"），要解释这如何服务 emic/etic、Results/Discussion 分工或理论回收。
- 直接可借鉴的结构/写法（≥6 条），每条含：① 技巧名称 ② 这篇文章怎么做 ③ 为什么有效 ④ 我写论文时如何模仿。不要写内容启发，要写结构技巧和写法技巧。
