# Lit Verify · 文献验真（零容忍）

对一份文献清单逐条实查真伪。任何无法实查、字段对不上、查无此文的条目 → 剔除，**宁缺毋滥**。

## 调用约定（独立运行）

- 输入：任意格式的文献清单——检索候选表、粘贴的参考文献列表、BibTeX/RIS、论文引文节选、AI 生成的疑似文献。先把输入规整成统一表格（标题 / 作者+年份 / 期刊 / DOI 或 CNKI ID，缺字段留空）再逐条验。
- 输出：会话内验真报告 + 干净清单；项目内可落盘 `02_文献/验真报告.md`。
- 工具适配：以环境实际可用为准；全部不可用时给出每条的手动核查链接（doi.org / scholar / CNKI 检索式），不要凭印象判定真伪。

## 验真协议（每条三步）

1. **实查**：
   - 英文条目：用 DOI 去 doi.org / Crossref / OpenAlex 解析，核对标题 + 作者 + 年份 + 期刊；无 DOI 的用标题在 Google Scholar / OpenAlex / Crossref 精确检索；来自 WoS / ScienceDirect 的可用 `wos-paper-detail` / `sd-paper-detail` 复核。
   - 中文条目：用 `cnki-paper-detail` 抓详情页核对标题 / 作者 / 期刊 / 年份。
2. **被引数采集**：英文取 OpenAlex `cited_by_count` 或 Google Scholar 引用数；中文取 CNKI"被引"字段。记入条目（下游金字塔分层要用）。
3. **判定（零容忍）**：
   - ✅ 通过：全字段核对一致（容许大小写 / 缩写级差异）。
   - ⚠️ 存疑：找得到但字段有出入（年份差 / 期刊不符 / 作者顺序不同）→ 列出出入让用户定夺，默认按实查结果修正后保留。
   - ❌ 剔除：查无此文 / DOI 解析不到 / 标题拼接感明显且检索无命中。**直接剔除，不给"可能存在"的模糊结论。**

## 输出

1. **验真报告**：通过 X 条 / 修正 Y 条 / 剔除 Z 条 + 剔除原因逐条注明。
2. **干净清单表**（仅含通过与修正条目，新增"被引"列）：
   `| # | 标题 | 作者+年份 | 期刊 | DOI/CNKI ID | 被引数 | 验真状态(通过/修正) |`
3. **下一步建议**：清单要入库分层 → `lit-pyramid`；某维度剔除后不足 → 回 `pxyv-search` 补搜（给关键词建议）。
