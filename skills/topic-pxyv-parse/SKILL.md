# Topic PXYV Parse · 已有选题 → PXYV 四维拆分

不是选题教练，不重跑三阶段——把用户**已有的**选题结构化成 PXYV，并诊断缺口。

## 调用约定（独立运行）

- 输入：用户粘贴的选题描述 / 研究计划 / 摘要 / RP 段落，或一个文件路径。都没有 → 请用户用几句话说清「想研究什么、用什么视角、在什么语境、图什么贡献」。
- 落盘：`选题.md`（顶部含锚块，与选题工作流产出格式一致，下游无感），默认当前工作目录；用户只要会话内结果就不落盘。
- 产出语言：项目 CLAUDE.md 有设定则从之，否则跟随用户。

## 协议

**1 · 判型**：先判断这个选题最贴近哪种模式（①理论诠释 ②假设检验 ③概念建构 ④方法驱动），一句话理由，写入锚块 mode 字段。

**2 · 四维拆分**（每一维给出拆分 + 依据原文的哪句话；拆不出来就标 [缺失]，不要硬编）：
- **P 理论视角**：用的什么理论 / 范式 / 方法范式？提醒：P 不是访谈 / 田野等数据方法。
- **X 现象**：研究对象是什么？学界可能的标准术语是什么（给 1-2 个候选）？
- **Y 语境**：时空 / 人群 / 行业 / 制度语境。
- **V 贡献**：这个研究图什么？按贡献阶梯初步定级（L1 语境扩展 / L2 机制条件 / L3 构念方法 / L4 假设挑战），并给 so-what 双句草案（① 理论/领域增量句 ② 现实利害句——具体人群/机构，不接受"全社会"）。

**3 · RQ / 标志产出提取**：从原文提取（或按 mode 体例草拟 2-3 版让用户选）：mode ① RQ/Thesis；mode ② H 假设链方向；mode ③ 原创概念 + 开放 RQ；mode ④ Aims/直接陈述。

**4 · 缺口诊断**（一张表）：`| 维度 | 拆分结果 | 强/弱/缺失 | 建议 |`——弱或缺失的维度给具体建议：P 弱 → 建议跑对应 mode 选题 skill 的 A.3；V 没校准 → 建议跑阶段 B 校准（B.4 V 空白检验 + B.5 近邻对位）；近邻空缺 → 建议用 `lit-verify` + 检索补 5 近邻。**不替用户补编内容，只指路。**

**5 · 产出锚块 + 正文**：

```
<!-- workbench
mode: [1-4]
title: [一句话标题]
P: … | X: … | Y: … | V: …
V-level: [L1-L4 · 初判，未经 B 校准的标注 (未校准)]
SW: [so-what 双句压缩成一句]
journal: [有则填，无则 TBD]
RQ: [研究问题一句话]
neighbors: [有则填；无则留 TBD——未经近邻对位]
-->
```

锚块下写人类可读正文：四维拆分表 + RQ + so-what 双句 + 缺口诊断表。

## 硬约束

- 只拆用户给的内容，缺位标 [缺失] / TBD，**不编造**理论、期刊、近邻文献。
- 未经阶段 B 校准的字段（V-level / neighbors）必须带「未校准」标记，避免下游误当已验证结论。
- 拆完一句话总结：哪几维立得住、哪几维要回炉，回炉走哪个 skill。
