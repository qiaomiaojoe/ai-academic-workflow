# Article Framework · 反推文章框架（5 Part + 可视化）

基于 Model Paper 拆解报告反推新论文框架。产出 ① 文章框架.md（5 Part 骨架）② 文章框架-part2.html（可视化）。扩写成全文交给 `fulltext-draft`。

## 调用约定（独立运行）

- 输入按"用户直给 > 项目文件 > 降级"解析：
  1. **选题背景（必需，可口头给）**：`选题.md`（选题产出）或 RQ + PXYV 一段话（最少：RQ + 理论视角 + 研究对象）。
  2. **Model Paper 拆解报告（必需）**：`Model-Paper-拆解-*.md`（`model-paper-deconstruct` 的产出；多份先让用户确认用哪份）或用户给的路径。没有 → 先跑 `model-paper-deconstruct`（用户同意也可按通用 IMRaD 降级，Part 5 跳过）。
  3. **综述初稿（强烈建议）**：`综述初稿.md`（literature-analysis 的产出）或用户给的文本。没有 → Part 4 跳过，引用候选全标 [TBD-citation]，交付说明注明"综述缺失，建议先跑 literature-analysis"。
  4. **研究设计初稿（可选）**：`研究设计初稿.md` 存在则方法论节直接采用其方法、结构与引用并据此填引用预算；不存在按通用 IMRaD 占位，不因缺它而停。
- 落盘（默认当前工作目录，文件名固定不加日期后缀）：`文章框架.md` + `文章框架-part2.html`。
- 输出语言：项目 CLAUDE.md 或用户指定（可视化 HTML 仅中文）。

## 关键约束

- 每个段落骨架字段固定：功能 / 引用角色 / 动静虚实归属 / 引用候选。
- 引用配置必须和拆解报告的"三种引用角色"对得上；综述初稿各段必须能找到归属（Part 4 强制）。

## Part 1 · 全文层面框架

- 1.1 新论文四象限定位：用 PXYV 填四格（每格 3-5 条具体内容）
- 1.2 虚实转换三过程：本文哪一节做 conceptualisation / operationalisation / re-conceptualisation
- 1.3 三种引用角色全文配置：锚定候选 1-2 / 定位候选 2-3（覆盖 P 票根 + X×Y 票根）/ 框架候选 1（必须从综述初稿或 5 近邻里出，不自造）；PV 双重可仿 Model Paper 做双框架引用 + 整合段
- 1.4 承重 vs 装饰比例：目标 25-30% 承重 / 70-75% 装饰；全文预估引用 50-65 条（按 11000 字；短论文减半）
- 1.5 全文 Section → 象限功能图：节数 = 拆解产出 4 的实测节数（不预设）；每节填标题候选 / 主象限 / 学术引用数预估 / 政策·媒体引用数预估 / 功能一句话

## Part 2 · 各节段落骨架

节数与每节段数 = 拆解报告实测；节标题按新论文 P/X/Y/V 调整，保留节功能逻辑。⚠️ 不要假设一定有独立 Lit Review 节——Model Paper 把综述嵌入 Intro 的，新论文也可不设。每段：

```
## Sec X · [节标题]（对应 Model Paper 的 Sec Y · 功能：[一句话]）
### § 1 · [段落功能名]
| 字段 | 内容 |
|------|------|
| 功能 | 一句话说这段干什么 |
| 引用角色 | 锚定/定位/框架 + 装饰/承重 |
| 引用候选 | 具体文献（来自综述初稿 + 选题 5 近邻） |
| 动静虚实 | 主象限（允许"动实→动虚"转换标签） |
```

## Part 3 · 引用预算表

按新框架实际节名：`| Section | 学术引用数 | 政策/媒体引用数 | 备注 |`——Intro 含嵌入式综述则引用量最高；方法论节中等、政策 0（有研究设计初稿则直接采用）；发现/结果节极低或 0（⚡ 反直觉）；讨论节 3-5（含新进场）；结论 1-3；合计学术 40-55 + 政策/媒体 10-15；<8000 字减半。

## Part 4 · 与综述初稿的衔接表（⭐ 关键；综述缺失时跳过）

`| 综述初稿来源 | 映射到新框架位置 | 可直接复用 / 要重写 | 改写指引 |`（S1 经验段→部分复用；S2 理论段→大部分复用；S3 Gap→重写、移到 Intro 等；具体节号从拆解产出 4 读取）。末段加"综述初稿覆盖率评估"：覆盖百分比 + 还缺哪些类型文献（直接说是否要补搜）。

## Part 5 · 与 Model Paper 的结构对照表

`| 新框架节标题 | Model Paper 对应节 | 同构点 | 差异点 | 借鉴点 |` + 末段"仿写边界声明"：结构可抄（段落数 / 引用角色配置 / 动静虚实分布 / Section 顺序）；argument 不抄（每段说什么 / 用什么数据 / 引哪些文献）。

## 可视化（文章框架-part2.html）

仅中文。每段 4 字段：① 段落标题·功能名 ② 象限 tag（静实/静虚/动实/动虚，可转换标签）③ 引用角色 tag（锚定·装饰 / 定位·部分承重 / 框架·承重 / 无引用）④ 功能一句话 + 引用候选。样式：背景 #fafaf7 / 纸面 #fff / 文字 #1a1a1a / 辅助 #6b6b6b / 边线 #e0ddd6；象限配色 静实 #fde2e4 / 静虚 #d4e4f5 / 动实 #fce8c7 / 动虚 #d0ebd6；引用角色配色 锚定 #c7b8d8 / 定位 #f5b97f / 框架 #e85d75（白字）；header + legend + sticky nav；每节 `<section class="section">`（含对应 Model Paper 节的 mirror 行），每段 `<div class="para">`（左 4px 边框，para-head + fields）；PingFang SC 系，行高 1.7，max-width 1200px，CSS 全内联，无 JavaScript。
