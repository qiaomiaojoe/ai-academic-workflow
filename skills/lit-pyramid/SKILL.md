# Lit Pyramid · 金字塔分层（Classic / Key / Supporting）

对**已验真**的清单做三件事：金字塔分层、对话关系判定、覆盖配额检查。真伪 / 相关性初筛不在本 skill（先跑 `lit-verify`）。

## 调用约定（独立运行）

- 输入：已验真的文献清单（含被引数；来自 `lit-verify`，或用户给的 Zotero 集合 / 表格）+ 本研究 RQ（用户给一句话，或读项目 `01_选题/选题.md`）。清单没有被引数 → 先补采（OpenAlex / Scholar / CNKI）。
- 参数（默认可覆盖）：Key 篇数默认 5；Supporting 每节点配额默认 = 余量/4。
- 输出：会话内分层表；项目内可落盘 `02_文献/文献清单.md`（分层版）。

## 金字塔三层硬指标

- **Classic（1 篇）** 四条全满足：① P 维度核心概念的命名者/奠基者（同领域 ≥3 篇综述把它列为概念出处）② 总被引 ≥500（社科）或 ≥1000（自然科学/CS）③ 发表 ≥10 年仍在被引（近 2 年仍有 ≥10 次新引用）④ 不受发表年限限制。
- **Key Texts（N 篇）** 三条全满足：① 发表在 P 所在领域 top-tier 期刊（表格注明期刊名 + 位次依据：SSCI Q1 / 北大核心 / 影响因子 / CNKI 高被引）② 与本研究 RQ 存在**直接对话关系** ③ 近 5 年年均引用 ≥10。整组 Key 必须覆盖 P / X / V 三维，每维 ≥1 篇（Y 通常无 Key）。
- **Supporting（~余量）**：① 提供具体证据（数据 / 案例 / 方法对照 / 政策文本 / 行业报告均可）② 按 P/X/Y/V 四节点分配：P/X ≥5，Y/V ≥3 ③ 同行评审优先；权威机构报告可作 Y 节点补充。

## 三机制检查

1. **分层归位**：每篇明确判到 Classic / Key / Supporting / 拒（四选一），注明分层证据（Classic 给引用数 + 概念出处证据；Key 给期刊名 + 年均引用 + 对话关系；Supporting 给证据类型）；不满足任一层硬指标 → 拒并注明具体指标。
2. **对话关系（Key 必答）**：这篇是在**支持、扩展、还是反驳**本研究 RQ 的某个立场？一句话说明。仅"主题相关"不构成对话 → 降级 Supporting。
3. **覆盖配额**：P ≥5 / X ≥5 / Y ≥3 / V ≥3？Key 覆盖 P/X/V？任一未达标 → 输出「维度缺口表」（缺哪些 + 补搜关键词建议）。**不允许用编造或勉强的文献凑配额**，让用户决定是否回 `pxyv-search` 补搜。

## 输出

对每篇入选文献额外抽取一句话核心论点（≤30 字，供下游可视化悬停提示）。

`| # | 标题 | 作者+年份 | 金字塔层 | PXYV 节点 | 分层证据 | 对话关系（仅 Key） | 核心论点(≤30字) | DOI/CNKI ID | 被引数 |`

表后附：被拒清单及原因 + 维度缺口表（若有）。**下一步建议**：拿分层清单去 `lit-pdf-zotero` 获取 PDF 并入库 Zotero。
