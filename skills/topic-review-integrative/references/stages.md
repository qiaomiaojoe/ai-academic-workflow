# 综述③ 整合式综述 · 三阶段详细步骤

方法锚：Snyder (2019), *Journal of Business Research*, 104, 333-339.
范例锚：Rodell, Breitsohl, Schröder & Keating (2016). Employee volunteering: A review and framework for future research. *Journal of Management*, 42, 55-84.

**范例预览（一次性端给用户看，作全貌参照）**：Rodell et al. (2016) 面对"员工志愿"零散的文献 → 批判性整合 → 提出一个**未来研究框架**，而不只是汇总。这就是整合式综述的样子：批判、重构、产出一个新框架 / 类型学。

## STAGE A · 挖掘（理清 研究对象 / 范围 / 问题）

> 三型通用。教练**先给 2-3 个草案**让用户选 / 改，一步一问。**不要求用户诊断"文献是不是很乱"**——文献的张力 / 缺口由教练在 A.4 帮点出。本型 A.3 侧重把问题往**"要造什么新东西"**上引。

- **A.1 研究对象**：据用户输入给 2-3 个"到底要整合什么"的更聚焦界定（一个概念群 / 一片现象 / 多个各自为政的传统），用户选 / 改。
- **A.2 情景 / 范围**：给 2-3 个范围候选——现有理论 / 概念在哪里不够用、涉及哪些传统、大致边界（整合式可含书、跨传统）。
- **A.3 研究问题（雏形）**：给 2-3 个问题草案，**引导落到"能不能整合出一个新框架 / 概念 / 类型学"**。顺带判断：若用户其实只想理清现状 / 给议程 → 提示更适合 `topic-review-semisystematic`；只想综合某效应 → `topic-review-systematic`。
- **A.4 为什么值得做 + so-what**：帮用户想清"为什么现在需要这次整合、谁会因此不同"。**教练主动点出这片文献可能的张力**（如老框架跟不上新现象、概念混乱需要重新定义、多个传统各自为政缺一个统一框架）作为佐证——教练的判断输入，不是用户门槛。产出 ① 理论增量句 ② 现实利害句（落到具体人群 / 机构）。
- **A.5 挑战**：你是真要造新东西，还是其实只想汇总？（只想汇总 → 劝退或改 `topic-review-semisystematic`）已有整合式综述是否已造过这个框架？你的差异在哪？
- **A.6 输出** → 汇总写入 `选题-A-mining.md`（中间产物 · 断点续跑用；中英对照则同文件双语并排）。

## STAGE B · 校准（整合式特化）

- **B.1 你要造什么**：给 2-3 个"贡献形态"候选（一个新框架 / 类型学 / 重新定义的概念），用户选。**这是整合式的灵魂，先定死。**
- **B.2 成熟 vs 新兴**：判断这个主题是成熟（→ 重构 / 挑战现有理论）还是新兴（→ 初步概念化），据此定语料策略与野心。
- **B.3 批判角度**：给 2-3 个批判切口——你要挑现有文献哪个假设 / 盲点（problematization, Alvesson & Sandberg 2011, AMR 36(2): 247-271）。整合式不是罗列，是批判后重构。
- **B.4 语料策略**：整合式可含书、跨传统、非穷尽——定"创造性取样"的逻辑与边界（仍要透明，让读者能判断）。下游数据分析走定性 / 概念分析。
- **B.5 近邻综述对位**：让用户列 2-3 篇已有整合式综述 / 理论文章（真实、可核对），说你的新框架跟它们什么关系（超越？调和？替代？）。
- **输出** → 写入 `选题-B-calibration.md`。

## STAGE C · 输出

### C.0 综述选题压力测试（Snyder 质量四维 depth / rigor / replicable / useful + §5 坑；整合式重点——Snyder 最强警示）
- 通用四问：① 贡献是不是真"新"（不是描述性汇总 / 词云 / 引文统计）？② 范围有没有砍太狠？③ 类型-问题-分析法三者匹配吗？④ 透明度够不够？
- 整合式特化：**这是"真整合"还是"披着整合皮的汇总"？** 新框架站得住吗、能一句话说清它新在哪吗？批判角度成立吗？
- 不过关 → 打回 Stage B（尤其 B.1 / B.3）。

- **【综述规格块】**：综述类型 integrative / 要造什么（新框架/类型学/概念）/ 成熟 or 新兴判断 / 批判角度 / 语料策略 / 预期贡献 + "新在哪" / 报告规范 Torraco 2005 / 分析走向（定性/概念分析 → 数据分析步骤）。
