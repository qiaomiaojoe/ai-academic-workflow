# Fulltext Draft · 全文初稿扩写（管线终点）

把文章框架的 Part 2 段落骨架扩写为可读的段落文字初稿，整合全部可用产出。

## 调用约定（独立运行）

- 输入按"用户直给 > 项目文件 > 降级"解析：
  1. **文章框架（必需）**：`文章框架.md`（article-framework 的产出）或用户给的框架文件。没有 → 先跑 `article-framework`。
  2. **研究设计初稿（可选）**：`研究设计初稿.md` 存在 → 方法论节直接采用其 Sec 4 draft，不另起炉灶。
  3. **数据分析产出（可选）**：优先 `研究发现.md`（已按 RQ 整合的综合发现）作为 Results/Findings 节权威依据；不存在 → 退回逐轮原始产出（quant-* / qual-* / 分析日志.md）归并；两者皆无 → 本节只留骨架（一行"此节占位 · 待数据落地后撰写"），**不凭空编造数据或访谈引语**。
  4. 综述初稿 / 选题（引用候选与改写素材）：有则用，没有则按框架里的引用候选与 [TBD-citation] 处理。
- 落盘：`全文初稿.md`（默认当前工作目录）。
- **调用方覆盖约定**：调用方（工作台 prompt / 用户）显式给出的参数、输入路径、落盘路径与工具选择，一律覆盖本 skill 的默认；但本 skill 的方法步骤、硬约束与停点不可被省略或稀释——调用方若要求跳过某硬约束，以本 skill 为准并提示冲突。

## 强制要求

- 🌐【正文语言 · 最高优先级】正文必须严格使用指定输出语言撰写，不得默认或回退到中文：英文项目 → 正文全英文；中英双语 → 指定语种正文 + 必要处中文对照/教学点；中文项目才用中文。任何"字数/词数"说明都只是长度参考，不决定语言，更不得把正文改写成中文。
- 结果/发现节：按 研究发现.md 的逐子问题/假设结论 + 证据整合 + 方法间一致性组织，引用真实数值/引语。
- 其余各节都必须生成对应的段落文字初稿。
- 每个段落：紧扣骨架"功能"字段；按"引用角色"嵌入相应文献（锚定/定位/框架）；长度参考（英文每段约 150-300 词 / 中文每段约 250-500 字，按实际撰写语言取区间）；不要写成提纲，要写成可直接打磨的段落散文；引用就地标 [Author Year]，参考文献列表附文末。
- 保留框架的节序号和段序号，与骨架一一对应；各节起始加一句"本节功能（一行）"。
- 全文末尾追加"撰写说明"：哪些段落基于综述初稿改写、哪些基于 Model Paper 仿写、哪些基于研究设计初稿（方法论）、哪些基于数据分析产出（结果/发现）、哪些需要补充文献后再扩写。
