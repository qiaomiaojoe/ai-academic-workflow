# Fulltext Draft · 反推框架 + 全文初稿（三产出）

一次性产出三份文件：① 文章框架.md（5 Part 骨架）② 全文初稿.md（全文段落初稿）③ 文章框架-part2.html（可视化）。

## 单品调用约定（输入解耦）

五路输入，按优先级"用户直给 > 项目文件 > 降级"逐一解析：
1. **选题背景（必需，可口头给）**：项目 `01_选题/选题.md`（PXYV + RQ + 5 近邻）或用户直接给 RQ + PXYV 一段话。两者皆无 → 问用户要（最小：RQ + 理论视角 + 研究对象一句话）。
2. **综述初稿（强烈建议）**：项目 `02_文献/综述初稿.md`（Section 1/2/3）或用户给的综述文本/文件。没有 → Part 4 衔接表跳过，引用候选全部标 [TBD-citation]，并在交付说明里注明"综述缺失，建议先跑 literature-analysis"。
3. **Model Paper 拆解报告（必需）**：项目 `03_框架/Model-Paper-拆解-*.md`（通配符匹配；多份先列出文件名让用户确认用哪份）或用户给的路径。没有 → 提示先跑 `model-paper-deconstruct` skill（也可以用户同意后按通用 IMRaD 结构降级，Part 5 对照表跳过）。
4. **研究设计初稿（可选）**：项目 `03_框架/研究设计初稿.md` 存在 → 方法论节直接采用它的方法、结构与引用，并据此填引用预算；不存在 → 方法论节按通用 IMRaD 占位，不要因缺它而停。
5. **数据分析产出（可选）**：优先读项目 `04_数据/研究发现.md`（已按 RQ 整合的综合发现）作为 Results/Findings 节权威依据；不存在 → 退回 `04_数据/` 逐轮原始产出（quant-* / qual-* / 分析日志.md）自行归并；两者皆无 → 结果节按占位处理，**不要凭空编造数据或访谈引语**。

**输出落盘降级**：在项目里 → `03_框架/文章框架.md` + `03_框架/全文初稿.md` + `03_框架/文章框架-part2.html`；不在项目里 → 当前目录同名三件。

**输出语言**：项目 CLAUDE.md 或用户指定；见下方最高优先级语言规则。

## 关键约束

- 每个段落骨架的字段固定：功能 / 引用角色 / 动静虚实归属 / 引用候选
- 引用配置必须和 Model Paper 拆解报告的"三种引用角色"对得上
- 综述初稿的各段必须能找到归属（Part 4 衔接表强制要求；综述缺失时跳过并注明）
- 骨架仅作为扩写的脚手架，最终交付物包含骨架 + 文字初稿 + 可视化三件

## 产出 1 · 文章框架.md（5 个 Part）

**Part 1 · 全文层面框架**：
- 1.1 新论文四象限定位：用选题的 PXYV 填四格（每格 3-5 条具体内容）
- 1.2 虚实转换三过程：本文哪一节做 conceptualisation / operationalisation / re-conceptualisation
- 1.3 三种引用角色全文配置：锚定引用候选 1-2 个 / 定位引用候选 2-3 个（覆盖 P 票根 + X×Y 票根）/ 框架引用候选 1 个（必须从综述初稿或 5 近邻里出，不自造）。若 PV 双重，可仿 Model Paper 做双框架引用 + 整合段设计
- 1.4 承重 vs 装饰比例：目标 25-30% 承重 / 70-75% 装饰；全文预估引用总量 50-65 条（按 11000 字算；短论文减半）
- 1.5 全文 Section → 象限功能图：节数 = Model Paper 拆解产出 4 的实测节数（不预设）；每节填：标题候选（按输出语言）/ 主象限 / 学术引用数预估 / 政策·媒体引用数预估 / 功能（1 句话）

**Part 2 · 各节段落骨架**：
- 节数 = Model Paper 产出 4 实测节数；每节段数参考 Model Paper 对应节实测段数；节标题按新论文 P/X/Y/V 调整，保留 Model Paper 的节功能逻辑
- ⚠️ 不要假设一定有独立的 Lit Review 节——若 Model Paper 把综述嵌入 Intro（如 4-strand 结构），新论文也可不设独立综述章节
- 每段格式：

```
## Sec X · [节标题]（对应 Model Paper 的 Sec Y · 功能：[一句话]）
### § 1 · [段落功能名]
| 字段 | 内容 |
|------|------|
| 功能 | 一句话说这段干什么 |
| 引用角色 | 锚定/定位/框架 + 装饰/承重 |
| 引用候选 | 具体文献（来自综述初稿 + 选题 5 近邻） |
| 动静虚实 | 主象限（允许"动实→动虚"这类转换标签） |
```

**Part 3 · 引用预算表**（按新框架实际节名，不预设 Sec 1-9 标签）：
`| Section（实际节标题）| 学术引用数 | 政策/媒体引用数 | 备注 |`——Intro 节若含嵌入式综述引用量最高；方法论节中等、政策引用 0（有研究设计初稿则直接采用其引用）；发现/结果节极低或 0（⚡ 反直觉：findings 节几乎不引学术）；讨论节 3-5（含新进场理论资源）；结论节 1-3。全文合计学术 40-55 + 政策/媒体 10-15，总量 50-65；论文 < 8000 字按此减半。

**Part 4 · 与综述初稿的衔接表**（⭐ 关键；综述缺失时跳过）：
把综述初稿各段映射到新框架中承接综述/理论功能的节（具体节号从 Model Paper 拆解产出 4 读取，不预设）：
`| 综述初稿来源 | 映射到新框架位置 | 可直接复用 / 要重写 | 改写指引 |`（S1 经验段 → 部分复用；S2 理论段 → 大部分复用；S3 Gap → 重写、Gap 移到 Intro 等）
末段加"综述初稿覆盖率评估"：覆盖了综述/理论节的多少百分比？还缺哪些类型文献？（直接说是否需要补搜）

**Part 5 · 与 Model Paper 的结构对照表**：
`| 新框架节标题 | Model Paper 对应节 | 同构点 | 差异点 | 借鉴点 |`
末段加"仿写边界声明"：结构可抄（段落数 / 引用角色配置 / 动静虚实分布 / Section 顺序）；argument 不抄（每段具体说什么 / 用什么数据 / 引哪些文献）。

## 产出 2 · 全文初稿.md（终点 · 承接全部产出）

基于 Part 2 骨架扩写为可读的段落文字初稿，整合全部可用输入。

⚠️ 强制要求：
- 🌐【正文语言 · 最高优先级】正文必须严格使用指定输出语言撰写，不得默认或回退到中文：英文项目 → 正文全英文；中英双语 → 指定语种正文 + 必要处中文对照/教学点；中文项目才用中文。任何"字数/词数"说明都只是长度参考，不决定语言，更不得把正文改写成中文。
- 方法论节：有研究设计初稿 → 直接采用其 Sec 4 draft 充实本节，不另起炉灶。
- 结果/发现节：优先据 研究发现.md 撰写（按其逐子问题/假设的结论 + 证据整合 + 方法间一致性组织，引用真实数值/引语）；无则退回逐轮原始产出归并；两者皆无 → 本节只保留骨架（一行"此节占位 · 待数据落地后撰写"），不编造。
- 其余各节都必须生成对应的段落文字初稿。
- 每个段落：紧扣骨架"功能"字段；按"引用角色"嵌入相应文献；长度参考（英文每段约 150-300 词 / 中文每段约 250-500 字，按实际撰写语言取区间）；不要写成提纲，要写成可直接打磨的段落散文；引用就地标 [Author Year]，参考文献列表附文末。
- 保留 Part 2 的节序号和段序号，与骨架一一对应；各节起始加一句"本节功能（一行）"。
- 全文末尾追加"撰写说明"：哪些段落基于综述初稿改写、哪些基于 Model Paper 仿写、哪些基于研究设计初稿（方法论）、哪些基于数据分析产出（结果/发现）、哪些需要补充文献后再扩写。

## 产出 3 · 文章框架-part2.html（可视化）

把 Part 2 段落骨架渲染为独立 HTML（教学/讨论展示）：
- 仅中文版（不做双语切换）。每段只保留 4 字段：① 段落标题·功能名 ② 象限 tag（静实/静虚/动实/动虚，允许转换标签）③ 引用角色 tag（锚定·装饰 / 定位·部分承重 / 框架·承重 / 无引用）④ 功能一句话 + 引用候选。
- 样式：背景 #fafaf7、纸面 #ffffff、文字 #1a1a1a、辅助 #6b6b6b、边线 #e0ddd6；象限配色 静实 #fde2e4 / 静虚 #d4e4f5 / 动实 #fce8c7 / 动虚 #d0ebd6；引用角色配色 锚定 #c7b8d8 / 定位 #f5b97f / 框架 #e85d75（白字）。
- 顶部 header（论文标题副标题 + 节数/段数/仿写锚说明）+ legend（象限 + 引用角色图例）+ sticky nav（各节锚点）。
- 每节一个 `<section class="section">`（section-head：节号 + 标题 + 对应 Model Paper 节的 mirror 行 + paragraphs 容器）；每段一个 `<div class="para">`（左侧 4px 边框，para-head：§ N + 段落功能名 + tag；fields：功能 / 引用候选两行）。
- 字体 PingFang SC / Source Han Sans SC 系；行高 1.7；max-width 1200px；所有 CSS 内联，无外部依赖、无 JavaScript。
