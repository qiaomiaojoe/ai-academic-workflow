# Research Gap Scan · Gap 识别（组合扫描 + 三问反驳）

基本思路：gap = 选题关键词 P、X、Y 之间的**新组合**。所以必须先知道已有组合，再找空白——不允许凭印象报 gap。

## 调用约定（独立运行）

- 输入：① 选题 PXYV（用户给 / 项目 `01_选题/选题.md` / 一句话选题先经 `topic-pxyv-parse`）② 文献覆盖证据（三选一：知识模块清单 / Zotero 集合 / 文献清单——没有任何文献证据时明确降级为"检索式抽查"模式：对每个候选格子现场检索近 3 年命中数作为覆盖证据，并注明证据强度弱于全库扫描）。
- 落盘：项目内写入 `02_文献/综述蓝图.md` 的 Section 3（已有蓝图则填入，没有则单独落 `02_文献/gap-扫描.md`）；项目外 `./gap-扫描.md`。

## 协议

1. **展开维度**：P 的近邻概念、X 的操作化变体、Y 的操作化变体（各 3-5 个）。
2. **组合表**：枚举文献已覆盖的 (X,Y) / (P,X) / (P,Y) 组合（依据模块清单 / 集合内容 / 现场检索），一张表标明每格覆盖强度（强/弱/空白）。
3. **候选 gap**：空白 / 弱覆盖的格子。
4. **三问反驳（防伪 gap，每个候选必过）**：
   a. 这是真空白，还是已被验证为无意义 / 已被否证？
   b. 本研究 RQ 是否正好落在这个格子？
   c. 文献里有没有人明确呼吁过这个方向？
5. **归类**：通过三问的按类型归类——empirical / contextual（人群·情境）/ theoretical（理论冲突）/ methodological。
6. **输出**：核心 gap 3-5 个（具体且通过三问，每个附组合表出处与三问答案）+ 本研究 RQ 如何回应 + 定位陈述（1 段）。

## 硬约束

- 每个 gap 必须能回溯到组合表的具体格子 + 覆盖证据；"研究不足""值得进一步探讨"这类无格子依据的表述不算 gap。
- 三问任一答不过 → 剔除，列入"伪 gap 清单"并注明被哪一问刷掉。
- 证据基础是抽查模式时，每个 gap 标注「证据：检索式抽查」，建议用完整文献库复扫。
