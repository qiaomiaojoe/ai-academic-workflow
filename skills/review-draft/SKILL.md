# Review Draft · 综述文章初稿扩写（综述文章工作台 · 管线终点）

把一篇**综述文章**扩写成可打磨的全文段落初稿。综述文章的 review 本身就是整篇论文——不是实证论文里的文献综述节（那用 `review-draft-assembly`）。成文结构由**报告规范**决定：系统综述→PRISMA、半系统/叙事→RAMESES、整合式→Torraco (2005)。

方法锚：Snyder, H. (2019). Literature review as a research methodology: An overview and guidelines. *Journal of Business Research*, 104, 333-339.

## 调用约定（独立运行）

- 输入按"用户直给 > 项目文件 > 降级"解析：
  1. **选题.md（必需）**：读**文件最顶部的机器锚块**，取 `review_type`（systematic / semi-systematic / integrative）与 `report_standard`（PRISMA / RAMESES / Torraco (2005)）**定成文结构**；读锚块下方【综述规格块】（RQ / 纳入排除 / 组织框架 / 预期贡献）+【PXYV 派生投影】+ 近邻综述表作为写作依据。缺 → 指路先去「综述文章工作台」阶段① 用 `topic-review-*` 产出选题.md。
  2. **研究发现.md（③ 综合结果）**：存在 → **综合 / 结果节**据其逐子问题结论 + 证据整合撰写（meta-analysis 报效应量与 CI、定性报主题与脉络、整合式报新框架）；不存在 → 退回逐轮原始产出（`综合-*` / 分析日志.md）归并；两者皆无 → 综合/结果节只留骨架（一行"此节占位 · 待③综合落地后撰写"），**不凭空编造效应量、主题或引语**。
  3. **语料库 + PRISMA 筛选记录（02_文献/）**：存在 → **方法节**的检索策略 / 纳入排除 / PRISMA 数量流（识别→去重→筛→纳入）+ 流程图据其撰写；缺 → 方法节按选题.md 的检索协议种子写，并在交付说明标注"检索执行记录待补"。
  4. 近邻综述表 / 已有综述：用于 Introduction 定位"本综述与既有综述的差异"。
- 参数：报告规范（默认跟随选题.md 锚块 `report_standard`，用户显式给则覆盖）、总字数目标（可选）、引用格式（默认 APA 7th）、输出语言（项目 CLAUDE.md 或用户指定）。
- 落盘（默认当前工作目录）：`综述文章初稿.md`（+ 双语 `-zh.md`）+ 同名 .docx + `引用核验报告.md`。**注意与 `review-draft-assembly` 的 `综述初稿.md`（实证论文综述节）区分——本 skill 产出的是整篇综述文章。**
- **调用方覆盖约定**：调用方（工作台 prompt / 用户）显式给出的参数、输入路径、落盘路径与工具选择，一律覆盖本 skill 的默认；但本 skill 的方法步骤、硬约束与停点不可被省略或稀释——调用方若要求跳过某硬约束，以本 skill 为准并提示冲突。

## 【报告规范决定结构】——先读选题.md 锚块 `report_standard`，按对应骨架成文

### systematic · PRISMA（系统综述）
1. **Introduction**：背景 + rationale + 明确的综述问题 / objectives（PICO 式若适用）+ 与既有系统综述的差异。
2. **Methods**：纳入 / 排除标准 · 信息源与检索式 · 筛选流程 · 数据抽取 · 综合方法（是否 meta-analysis）；给 **PRISMA 数量流**（识别 n → 去重 → 标题摘要筛 → 全文筛 → 纳入 n，各步排除数与理由）+ 流程图（文字描述或 mermaid）。
3. **Results**：纳入研究特征表 + 综合结果——有可比 effect size → meta-analysis（森林图逻辑、合并效应量 + 95% CI + 异质性 I²，据研究发现.md）；否则定性系统综述（按结局 / 主题归并证据）。
4. **Discussion**：证据总括 · 与既有结论对话 · 局限（发表偏倚 / 检索边界）· 结论。

### semi-systematic · RAMESES（半系统 / 叙事 / meta-narrative）
1. **Introduction**：主题背景 + 为何需要跨学科梳理 + 综述目的。
2. **Approach & Scope**：透明但非穷尽的检索与取样逻辑（哪些传统 / 时间窗 / 代表性取样），透明到读者能判断合理性。
3. **Synthesis**：按选题.md 的组织框架展开——按**主题 / 历时发展 / 研究传统 / meta-narrative** 组织，讲清领域如何演进、有哪些主题与张力（据研究发现.md 的主题综合）。
4. **Research Agenda**：从综合中提炼**研究议程**（真有洞见，非罗列）。
5. **Conclusion**。

### integrative · Torraco (2005)（整合式）
1. **Introduction**：主题重要性 + 为何现有理论不够用（成熟需重构 / 新兴需初步概念化）。
2. **Method**：语料选择与"创造性取样"逻辑（可含书、跨传统、非穷尽，但透明）+ 批判角度。
3. **Critical Analysis & Synthesis**：**批判性**评估现有文献（挑假设 / 盲点，problematization），按概念而非按文献逐篇组织——是重构，不是汇总。
4. **New Framework / Typology**（核心贡献）：据研究发现.md 呈现浮现出的**新框架 / 类型学 / 重定义概念**，一句话说清"新在哪"。
5. **Implications & Future Research**。

> 三型都以目标期刊的实际惯例微调节名与编排；报告规范给的是骨架与必备要素，不是死模板。

## 强制要求

- 🌐【正文语言 · 最高优先级】正文严格用指定输出语言撰写，不得默认或回退到中文：英文项目 → 全英文；中英双语 → 指定语种正文 + 必要处中文对照；中文项目才用中文。"字数/词数"只是长度参考，不决定语言。
- 【综合 / 结果节据实】按研究发现.md 的逐子问题结论 + 证据整合撰写，引用**真实**效应量 / 主题 / 引语；研究发现.md 缺失就占位，**绝不编造数据、效应量、主题或原文引语**。
- 【方法节据实】检索式 / 纳入排除 / 数量流据 02_文献/ 的 PRISMA 记录；无记录则据选题.md 检索协议种子写并标"待补执行记录"。
- 【每段成段落散文】不写成提纲；每段紧扣该节功能，就地标 inline citation [Author Year]（或 (Author Year, p.X)）；引用只用语料库里真实纳入的文献，不确定用 `[TBD-citation]` 占位。
- 各节起始加一句"本节功能（一行）"，便于打磨时对齐。

## 【引用核验（对照 Zotero 库 / 语料库清单）】

初稿写完后逐条核对 inline citation：用 `zotero_search_items` / `zotero_get_item_metadata` 确认每条引用的 Author/Year 真实存在于集合、拼写与年份无误（Zotero 不可用时对照语料库清单 / 本地 PDF 文件名核对）。产出独立的 `引用核验报告.md`（**不进初稿**）：每条引用 → 是否在库 → 有无出入。未能核实的引用，在初稿顶部"交付说明"里点名。

## 【参考文献表】

按指定引用格式（默认 APA 7th）在正文末尾生成"参考文献"表，覆盖正文所有 inline citation，仅列真实纳入语料库的文献。

## 【写作要求 · 反 AI 味（内联，不调用外部 skill）】

- 去掉空洞过渡词，禁用"值得注意的是 / 综上所述 / 不仅……而且"这类套话。
- 句长有变化，不要每句同一节奏。
- 具体优于抽象：能给机制 / 数字 / 效应量 / 对象就不泛泛而谈。
- 陈述性学术语气，不堆形容词。

## 【交付要求 · 初稿纯净】

- 正文零备注：不出现内部编号、Zotero key、生成过程说明。
- 风险提示集中放文档最顶部一个"⚠️ 交付说明"区块，仅限：③综合是否缺失 / 哪节证据偏薄、PRISMA 记录是否待补、未能核实的引用、缺页码的引用。除此之外正文是干净综述文章。
- inline citation 格式 (Author Year, p.X)。
- 输出语言按约定（中英双语 = 先英文 draft `综述文章初稿.md`，再中文译稿 `综述文章初稿-zh.md`，两稿都要 inline citation）。

## 【另存 Word】

md 初稿完成后转一份 .docx（用 pandoc 或 docx 能力，保留参考文献表与引用格式），与 md 同目录。中英双语项目：两份 md → 各转一份 docx。

## 流程

Step 0 读取（选题.md 锚块定型定规范 → 研究发现.md → 02_文献/ PRISMA 记录，缺的明说不假设）→ 按 `report_standard` 选对应骨架 → 逐节扩写成段落散文（综合/结果节据实、不编造）→ 引用逐条核验出 `引用核验报告.md` → 参考文献表 → 反 AI 味通读一遍 → 初稿纯净化（顶部⚠️交付说明）→ 转 docx。
