# Mode ① 理论诠释 · 三阶段详细步骤

范例锚论文：Manolev, Sullivan & Slee (2019). The datafication of discipline: ClassDojo, surveillance and a performative classroom culture. *Learning, Media and Technology*, 44(1), 36-51.

## STAGE A · 挖掘（6 步走）

A.1 故事五步
- 教学法：教练**先一次性把范例锚的完整五步端给用户看**（有全貌参照），**然后一步一问**，每步教练**先给 2-3 个草案候选**（基于用户的「问题/现象描述」推导），让用户选 / 改 / 重写
- 完整范例（Manolev 2019，一次性展示）：
  - 现象 = 一款叫 ClassDojo 的 App 进了澳大利亚小学，学生因课堂行为被加分/扣分
  - 主角 = 使用 ClassDojo 的小学教师、被打分的学生
  - 矛盾 = 声称"激励正向行为"，但学生开始竞争点数、有些反抗、有些表演性自律
  - 证据 = App 后台数据、教室观察、教师访谈、学生反应
  - 一幕 = 老师手机屏幕上的"班级实时排行榜"，学生抬头看自己的虚拟头像
- 一步一问（每步教练先给 2-3 个草案，用户选/改）：① 现象 ② 主角 ③ 矛盾 ④ 证据 ⑤ 一幕

A.2 内核（结构性张力）
- 范例：不是"App 好不好用"，是"数字化技术如何在日常空间里重新分配规训权力"
- 教练任务：基于 A.1，给用户 2-3 个内核草案（一句话），用户选 / 改
- so-what 起草（教练做，用户只确认）：每个草案自带一句"为什么值得做"；用户选定后，教练给出 so-what 双句草案 2-3 版——① 理论增量句：这项研究将修正/扩展学界对 [Y] 的什么认识；② 现实利害句：哪一类具体人群/机构的决策或实践会因此不同（不接受"全社会"）——用户选/改，双句进入阶段 C 的 V 表述与锚块 SW 字段
- 假设挑战识别（教练主动做）：检查用户的故事里是否有"现象与文献主流假设相抵触"的信号；若有，把「挑战假设版」作为候选之一端出来（problematization, Alvesson & Sandberg 2011, AMR 36(2): 247-271），说明它与「填空白版」的意义差别，用户选

A.3 P 视角（关键选择 — 基于用户的学科背景）
- 范例（教育社会学/批判教育技术，对应 Manolev 2019 论文实际引用）候选 P：
  - Datafication (Mayer-Schönberger & Cukier 2013；Bradbury 2018) — 关注日常被转化为数据 + 权力视角
  - Foucauldian discipline / governmentality (Foucault 1979) — 规训权力 + 治理术
  - Performativity (Ball 2003, 2013, 2015) — 表演性自我建构
  - EdTech-as-platform 分析 (Williamson 2017a, 2017b) — ClassDojo 作为校园社交媒体平台
  - 实际选：datafication 主导 + Foucauldian 权力框架 + performativity 解释学生反应（三者整合）
- 教练任务：给用户 2-3 个候选 P（每个简述概念 + 代表文献 + 为什么适合 ta 的内核），用户选 + 简述理由。提醒：P 不是访谈/田野等方法

A.4 PXYV 标题
- 范例：The datafication of discipline: ClassDojo, surveillance and a performative classroom culture
  - P = Datafication + Foucauldian discipline + performativity（三框架整合）
  - X = ClassDojo（school-based social media platform + 行为追踪 app）
  - Y = 当代学校教育中的纪律 / 行为管理实践
  - V = 揭示 ClassDojo 如何 intensify and normalise surveillance + 提出 "performative classroom culture" 作为新组合概念
- 教练任务：基于 P + X + Y 生成 2-3 个候选标题（格式 "The P of X in Y: towards V"），每个标注 P/X/Y/V；用户选/改

A.5 概念化句子（50-80 词，按产出语言；英文示范如下）
- 范例（参照 Manolev 论文 thesis 风格）："ClassDojo can be conceptualized as a datafication apparatus (Mayer-Schönberger & Cukier 2013) that operates through Foucauldian disciplinary mechanisms (Foucault 1979), that is, a digital tool that transforms classroom behavior into quantifiable data while simultaneously producing a self-regulating, performative subjectivity in young learners (Ball 2003)."
- 教练任务：给用户 2-3 个概念化句子草案（格式 "[X] can be conceptualized as [P]..., that is..."），用户选/改

A.6 输出 → markdown 保存到 `选题-A-mining.md`（中间产物 · 断点续跑用；中英对照则在同一文件内双语并排）

## STAGE B · 校准（B.1-B.4，之后做共用的 B.5）

B.1 P 校准 · "我的 P 是该学科主流吗？"
- 工具：Web Search + Zotero MCP
- 范例（Manolev 实际引用谱系）：搜 "datafication" + "education" 2014-2019 → 主流（Mayer-Schönberger & Cukier 2013 提出 datafication 一般定义，Williamson 2017 应用到 edtech 平台分析，Bradbury 2018 加入权力视角）
- 用户动作：搜 [你的 P] + [你的学科] 近 5 年 → 主流 / 部分主流 / 非主流 + 代表文献 2-3 篇

B.2 X 校准 · "我的 X 在学界用什么标准术语？"
- 工具：NotebookLM（导入 3-5 篇综述）或 Web Search
- 范例：ClassDojo 学界叫 "behavior management apps" / "classroom surveillance technologies" / "edtech for behavior tracking"
- 用户动作：问 "How is [我的 X] categorized in literature?" → 同义术语清单

B.3 Y 校准 · "我的 Y 在学界怎么表达？"
- 工具：Web Search
- 范例：搜 "performative classroom" + "Ball" → Ball 1990s 已提，但与"datafication"结合是 2010s 新组合
- 用户动作：搜时空/情境术语谱系 → 标准表达 + 是否需要调整

B.4 V 校准 · "我的 V 真的有 gap 吗？"（最重要 — 不读全文，只看空白）
- 工具：Zotero MCP + Web Search
- 范例：组合 P × X × Y 搜近 3 年 → Lupton 2015 提过"child datafication"但未做 ClassDojo 专项 → V 真有空白
- 用户动作：组合 P × X × Y 关键词搜近 3 年 → 结论 [有/没有/少量 N 篇]；只看显著空白

## STAGE C · 输出

阶段 C · mode ① 标志产出 = RQ 1-2 个 或 论点/Thesis 句
- 范例（Manolev 2019 实际是 thesis-driven 批判分析，无形式化 RQ；以下为论文原文 thesis）：
  - 论点 Thesis: "We argue that ClassDojo is a school-based technology of surveillance encouraging teachers to enact disciplinary practices underpinned by behaviourist principles that require intensified monitoring of students."（论文 Introduction 末）
  - Abstract 论点："ClassDojo's datafying system of school discipline intensifies and normalises the surveillance of students. Furthermore, it creates a culture of performativity and serves as a mechanism for behaviour control."
  - V 三层贡献：
    1. 理论：把 datafication + Foucauldian discipline + Ball performativity 三框架整合，应用到 ClassDojo 这一新对象
    2. 经验：基于公开线上材料（ClassDojo 网站 + 媒体报道 + 教师博客 + 公司声明数据）的关键解读 — 不是田野，是公开材料的批判分析
    3. 概念：提出 "performative classroom culture" 作为新组合概念（论文标题）
  - 方法（论文实际）："We draw on publicly available online material as a data source for this study."
- 用户产出（mode ① 包容两种形态，按研究气质选）：
  - 形态 A（RQ-driven 定性社科）：1-2 个 RQ，"How do X negotiate Y?" / "What kind of ... emerges from ...?"
  - 形态 B（Thesis-driven 批判分析，如 Manolev）：1 句论点 + V 三层贡献
  - 共同产出：V 三层贡献（理论 + 经验 + 方法/概念）+ 数据来源说明

教学锚句：理论诠释 mode 的灵魂是 P — 但 P 必须经过阶段 B 校准。RQ 或论点都是校准后 P 的具体落地。换 P，全篇变。

输出 → `选题.md`（唯一正式产出 · 顶部含 workbench 锚块）

# 三阶段共用机制块（所有 mode 通用）

## 【阶段 B 校准机制 · 每个动作按"检索式 → 判定标准 → 校准表"三步执行】

1) 检索式：教练先亮出本动作要跑的具体检索式（术语 + 年限 + 工具），用户可改后再执行
   （工具：Web Search / Zotero MCP / CNKI，按产出语言选；用户提供了对标论文时，从它的关键词、参考文献、施引文献出发作检索种子）
2) 判定标准（写死，不靠感觉）：
   - B.1 P 主流性：近 5 年 ≥3 篇目标领域（或目标期刊）论文以该理论/范式为主框架 → 主流；1-2 篇 → 可辩护（列出引文）；0 篇 → 换 P，或写明辩护理由再继续
   - B.2 X 术语：在 ≥2 篇检索命中文献中出现的表述才算"标准术语"，列同义术语清单
   - B.3 按本 mode 原有标准执行（量表有原始出处 / 样本数据可申请 / 案例有一手材料 / 基线可复现）
   - B.4 V 空白：三组检索式（P×X / X×Y / P×Y）近 3 年命中均 ≤5 篇、且无同 PXYV 组合 → 空白成立；任一组命中超限 → 列出最接近的文献，进 B.5 对位看差别
3) 每个动作产出校准表一行：维度 | 检索式 | 命中数 + 代表文献 | 判定 | 依据

## B.5 近邻对位（novelty 的实证检验 —— 锚块 neighbors 字段在这里产出）

- 近邻论文定义：与本选题在 P（理论/范式）、X（对象）、Y（语境/领域）三维中至少共享两维的已发表论文
- 检索机制：① 三组检索式 P×X / X×Y / P×Y（近 5 年，各取被引最高的若干篇）② 对标论文（若提供）的参考文献与施引文献 ③ 合并去重，按共享维度数排序取前 5，每篇标注共享的是哪两维
- 用户动作（教练先给对位句草案）：每篇一句对位 ——"它做了 ___，我的差别是 ___"
  5 句都写不出差别 → 选题撞车，回 A.4 调整 X 或 Y
- 产出：近邻表（引文 | 共享维度 | 对位句）

输出 → `选题-B-calibration.md`（中间产物 · 断点续跑用；含校准表 + 近邻表）

## C.0 选题压力测试（先做一轮，再产出标志成果）

- 四个受力面，教练对每面给出「最强的一条质疑」+「一条回应草案」，用户逐面判断（接受质疑并修改 / 采纳回应）：
  1. P-X 适配：这个理论/范式最解释不了现象的哪一面？需要补充 / 更换视角吗？
  2. 新颖性：对照 B.5 近邻表，撞车风险最大的是哪一篇？对位句立得住吗？
  3. 可行性：数据 / 样本 / 方法链条上最薄弱的一环是什么？
  4. 价值：教练检索目标期刊近 5 年是否发过同一对话的论文（检索式限定该刊 + P×X / X×Y 关键词）——命中 ≥1 篇 → 对话在该刊存在，列出篇目作证据；命中 0 篇 → 给出检索记录，建议换刊或调整定位。用检索证据说话，不要求用户凭空判断
- 本 mode 侧重：mode ①③ 重点测 1、2；mode ②④ 重点测 2、3（其余两面也要过，但从简）
- 不打分、不设门槛；4 组质疑 + 回应作为「压力测试记录」写入 选题.md 附录

## 【贡献阶梯 · 写 选题.md 时 V 必须标注定级】

- L1 复制 / 语境扩展：同一 P 换一个 X / Y
- L2 机制 / 条件：新中介、新调节、新边界条件
- L3 构念 / 方法：新概念、新测量、新方法
- L4 假设挑战：推翻或改写文献中被想当然的命题

（依据：L1-L3 ≈ Colquitt & Zapata-Phelan 2007, AMJ 50(6) 理论贡献五级量表的 1-2 / 3-4 / 5 级；L4 ≈ Alvesson & Sandberg 2011, AMR 36(2) problematization + Corley & Gioia 2011, AMR 36(1) revelatory。量表源自管理学：mode ①② 直接适用，mode ③ 类比适用，mode ④ 的方法贡献按"增量改进 → 已有方法新应用 → 新方法 → 新范式"类推）

教练给当前 V 定级，并问一次："往上推一级长什么样？代价是什么？" 推不推用户定。
