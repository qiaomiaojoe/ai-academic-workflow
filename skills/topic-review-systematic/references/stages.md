# 综述① 系统综述 · 三阶段详细步骤

方法锚：Snyder (2019), *Journal of Business Research*, 104, 333-339.
范例锚：Snyder et al. (2016). Identifying categories of service innovation. *Journal of Business Research*, 69, 2401-2408.

**范例预览（一次性端给用户看，作全貌参照）**：Snyder et al. (2016) 从"服务创新到底有几类"这个窄问题出发 → 定义严格纳入标准 → 系统收全文献 → 综合出一套服务创新**分类**。这就是系统综述的样子：问题窄、流程严、可复现、综合出定论。

## STAGE A · 挖掘（理清 研究对象 / 范围 / 问题）

> 三型通用。教练**先给 2-3 个草案**让用户选 / 改，一步一问。**不要求用户诊断"文献是不是很乱"**——文献的张力 / 缺口由教练在 A.4 帮点出。本型 A.3 侧重把问题**收窄到可回答**。

- **A.1 研究对象**：据用户输入给 2-3 个"到底要综述什么"的更聚焦界定（主题 / 概念 / 现象 / 变量关系 / 方法），用户选 / 改。
- **A.2 情景 / 范围**：给 2-3 个范围候选——放哪个领域、大致边界（学科 / 时间窗 / 纳入范围）。
- **A.3 研究问题（雏形）**：给 2-3 个问题草案，**引导往"某因素效应 / 两变量关系 / 某干预有没有用"上收窄**。顺带判断：若问题其实很宽、要理清脉络 → 提示可能更适合 `topic-review-semisystematic`；若要造新框架 → `topic-review-integrative`。
- **A.4 为什么值得做 + so-what**：帮用户想清"为什么现在需要这篇综述、谁会因此不同"。**教练主动点出这片文献可能的张力**（如结论互相打架、效应到底稳不稳没人量化汇总、证据分散）作为佐证——这是教练的判断输入，不是用户的门槛。产出 ① 理论增量句 ② 现实利害句（落到具体人群 / 机构，不接受"整个领域"）。
- **A.5 挑战**：这问题该用系统综述回答，还是其实该做一篇实证？已有系统综述是否已覆盖？你的差异在哪？
- **A.6 输出** → 汇总写入 `选题-A-mining.md`（中间产物 · 断点续跑用；中英对照则同文件双语并排）。

## STAGE B · 校准（系统综述特化）

- **B.1 收窄问题**：给 2-3 个更窄的、变量 / PICO 式改写（某因素对某结果的效应 / 两变量关系），用户选定可回答的具体问题。
- **B.2 纳入 / 排除标准**：给一版草案（年限 / 语言 / 研究类型 / 期刊 / 方法），**每条附一句理由**。提醒用户（Snyder）：这是决定质量最关键的一步；别为省事把范围砍太狠，否则漏掉相关研究、结论会偏。
- **B.3 检索协议种子**：产出"概念 × 同义词"检索矩阵 + 建议数据库（WoS / Scopus / …）+ PRISMA 流程（识别 → 筛选 → 纳入）。这份直接给下一步文献搜索用。
- **B.4 是否做 meta-analysis**：问用户手上是否有可比的 effect size。有 → 下游数据分析走**定量合成（meta-analysis）**；无 → 走**定性系统综述（qualitative systematic review）**。把结论记进规格。
- **B.5 近邻综述对位**：让用户列 2-3 篇已有系统综述 / meta-analysis（真实、可核对），逐一说你的增量在哪（更新？更宽？更严？新调节变量？）。
- **输出** → 写入 `选题-B-calibration.md`。

## STAGE C · 输出

### C.0 综述选题压力测试（Snyder 质量四维 depth / rigor / replicable / useful + §5 坑；系统综述重点）
- 通用四问：① 贡献是不是真"新"（不是描述性汇总 / 词云 / 引文统计）？② 范围有没有砍太狠（漏掉相关研究）？③ 类型-问题-分析法三者匹配吗？④ 透明度够不够让别人复现？
- 系统综述特化：纳入 / 排除标准是否明确到可复现？会不会因限得太死漏掉相关研究？
- 不过关 → 打回 Stage B 对应步骤。

### C.1 产出 `选题.md`
顶部机器锚块（见 SKILL.md 产出规则）+ 正文：
- **【综述规格块】**：综述类型 systematic / RQ / 纳入排除标准 / 检索协议种子（检索词·数据库·PRISMA 流程）/ 是否 meta-analysis / 预期贡献（效应证据 or 现状定论 + "新在哪"）/ 报告规范 PRISMA / 分析走向（有 effect size 定量 · 否则定性 → 数据分析步骤）。
- **【PXYV 派生投影】**：P=统摄问题的理论/效应关系；X=被检验的现象/干预；Y=领域+纳入边界；V=效应证据/现状定论。（只为下游 `pxyv-search` 兼容，由综述规格映射得出。）
- **近邻综述表**：B.5 的 2-3 篇，真实可核对。
