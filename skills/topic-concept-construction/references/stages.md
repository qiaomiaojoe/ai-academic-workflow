# Mode ③ 概念建构 · 三阶段详细步骤

范例锚论文：Manovich (2009). The Practice of Everyday (Media) Life: From Mass Consumption to Mass Cultural Production? *Critical Inquiry*, 35(2), 319-331.

## STAGE A · 挖掘（6 步走）

A.1 故事五步（聚焦"差异" — mode ③ 把"矛盾"换成"差异"）
- 教学法：教练**先一次性把范例锚的完整五步端给用户看**，**然后一步一问**，每步**先给 2-3 个草案候选**（基于用户的「问题/现象描述」推导），用户选 / 改 / 重写
- 完整范例（Manovich 2009，一次性展示）：
  - 现象 = 2000 年代中后期，YouTube、Flickr、博客平台兴起，普通人开始大规模创作和上传媒介内容
  - 主角 = 普通用户（不是专业生产者，也不是被动消费者）
  - 差异 = 20 世纪文化研究的核心二元"大众文化生产者 vs 大众消费者"失效了 — 普通人既不是生产者也不是消费者，是什么？
  - 证据 = YouTube 几亿小时用户上传视频、Flickr 几十亿张照片
  - 一幕 = 一个青少年用手机拍下日常 + 上传 YouTube + 引来几十万观看
- 一步一问（每步教练先给 2-3 个草案，用户选/改）：① 现象 ② 主角 ③ 差异（和过去/传统比，什么不一样了）④ 证据 ⑤ 一幕

A.2 内核 · 命名缺失
- 范例：de Certeau (1984) 的"日常生活实践"描述了消费者的创造性挪用，但 Web 2.0 是大规模文化生产 — 这是 de Certeau 没料到的新状态，需要新名字
- 教练任务：给 2-3 个"命名缺失"草案（这个状态在已有理论里叫什么？为什么不够？），用户选/改
- so-what 起草（教练做，用户只确认）：同 mode ①（理论增量句 + 现实利害句，双句进 C 的 V 表述与锚块 SW 字段）
- 假设挑战识别（教练主动做）：同 mode ①（problematization, Alvesson & Sandberg 2011, AMR 36(2): 247-271）

A.3 诠释路径候选 P
- 范例（媒介研究/文化研究/数字人文）候选：
  - 文化研究 + 软件研究（Manovich 自己开创）
  - 文化生产理论 (Bourdieu) — 偏精英 vs 大众
  - 参与式文化 (Jenkins) — 强调粉丝 vs 主流
  - 实际选：把 de Certeau "日常生活实践" 和 "软件研究" 结合
- 教练任务：给 2-3 个候选诠释路径（每个简述方法/代表理论家/能读出什么层次），用户选

A.4 原创概念命名（核心 V — mode ③ 的灵魂）
- 范例：把 "everyday life"（de Certeau）+ "media production" 缝在一起 → "The Practice of Everyday (Media) Life"
  加一个括号 (Media)，让旧概念发新声 — 这就是范例论文的标题，本身就是原创概念
- 教练任务：基于 P × X × Y，给 2-3 个原创概念命名草案
  - 格式："把两个既有词缝在一起"（一个来自理论传统、一个来自研究对象）
  - 技巧参考：加括号 / 形容词 + 名词 / 名词 + of + 名词
  - 用户选/改

A.5 PXYV 标题
- 范例：The Practice of Everyday (Media) Life: From Mass Consumption to Mass Cultural Production?
- 教练任务：给 2-3 个候选标题（用原创概念做主词），用户选/改

A.6 输出 → markdown 保存到 `选题-A-mining.md`（中间产物 · 断点续跑用）

## STAGE B · 校准（B.1-B.4，之后做共用的 B.5）

B.1 诠释路径校准
- 工具：Web Search
- 范例（Manovich）：de Certeau 1984 是文化研究核心著作（25000+ 被引）；与软件研究结合是 Manovich 自己开创 — P 既稳又有创新
- 用户动作：我的诠释路径在该传统的位置（主流 / 边缘 / 跨界）+ 2-3 篇代表文献

B.2 案例数据校准
- 工具：Web Search / 一手材料检索
- 范例：Web 2.0 平台数据 — YouTube 用户上传时长、Flickr 照片数、博客数 — 真实数据撑起"mass production"主张
- 用户动作：我选的 3-5 个案例是否有真实材料/数据支撑（不只是概念上存在）

B.3 概念谱系校准（mode ③ 核心）
- 工具：Web Search + Zotero MCP
- 范例："Everyday life" + "media production" 缝合前有谁尝试过？
  - Jenkins (2006) "participatory culture" — 只关注粉丝，不涵盖普通用户
  - Bruns (2008) "produsage" — 关注生产-使用融合，但偏技术中性
  - 没人从 de Certeau"挪用"传统延伸到大规模媒介生产 — 这是 Manovich 的位置
- 用户动作：我的原创概念 = 词A + 词B。词A 在什么传统？词B 在什么传统？谁接近过这个组合？我和他们的差别在哪？

B.4 概念能否撑起案例
- 工具：手动 / NotebookLM
- 范例：用 "Practice of Everyday (Media) Life" 分析 YouTube 普通用户视频 / Flickr 业余摄影 / 博客日常写作 — 三个案例都得到解释，概念立得住
- 用户动作：用这个概念分析每个案例 → 是否都能讲通。如果有案例无法被解释，要么换概念，要么换案例

## STAGE C · 输出

阶段 C · mode ③ 标志产出 = 诠释路径 + 案例切口 + 原创概念
- 范例（Manovich 2009 实际论证脉络，论文不分形式化 step；以下为论文真实结构）：
  - Section 1（Web 2.0 现象统计 + 提问）：列 Flickr / YouTube / MySpace / 移动手机用户规模数据；提出关键问题"What does this shift mean for how media functions?"，论文副标题就是 RQ："From Mass Consumption to Mass Cultural Production?"
  - Section 2（Tactics as Strategies）：引 de Certeau 1980《The Practice of Everyday Life》的 strategies vs tactics 区分；用 AMV（anime music videos）做主要 case study；引 Adorno & Horkheimer《Dialectic of Enlightenment》的 culture industry 批判作反问框架
  - 核心命题（论文原文 thesis）："the cultural tactics evolved by people were turned into strategies now sold to them" + "the logic of tactics has now become the logic of strategies" + "Web 2.0 paradigm represents the most dramatic reconfiguration of strategies/tactics relationship to date"
  - 原创概念 V：标题里的 "The Practice of Everyday (Media) Life" — 加括号让 de Certeau 经典发新声
- 用户产出：
  1. 诠释路径：列论证脉络（不必是 4-5 步，按论文需要的逻辑组织）—— 现象描述 / 既有框架引入 / 既有框架被挑战处 / 新概念提出 / 案例验证
  2. 案例（2-5 个，看论文体量）：每个案例的诠释切口 + 代表什么差异维度（Manovich 的主要 case 就是 AMV 一个，扩展到 Flickr/YouTube/Cyworld 等）
  3. 原创概念命名：把两个既有词缝在一起（一个来自理论传统，一个来自研究对象）
  4. 开放 RQ（论文用副标题即可，问号是论文体例）："From X to Y?" / "How does X get reconfigured under Y?"

教学锚句：Manovich 的标题《The Practice of Everyday (Media) Life》本身就是 V — 一个括号让 de Certeau 经典发出新声。"加括号"的命名技巧值得学。

输出 → `选题.md`（唯一正式产出 · 顶部含 workbench 锚块）

# 三阶段共用机制块（所有 mode 通用）

## 【阶段 B 校准机制 · 每个动作按"检索式 → 判定标准 → 校准表"三步执行】

1) 检索式：教练先亮出本动作要跑的具体检索式（术语 + 年限 + 工具），用户可改后再执行
   （工具：Web Search / Zotero MCP / CNKI，按产出语言选；用户提供了对标论文时，从它的关键词、参考文献、施引文献出发作检索种子）
2) 判定标准（写死，不靠感觉）：
   - B.1 P 主流性：近 5 年 ≥3 篇目标领域（或目标期刊）论文以该理论/范式为主框架 → 主流；1-2 篇 → 可辩护（列出引文）；0 篇 → 换 P，或写明辩护理由再继续
   - B.2 X 术语：在 ≥2 篇检索命中文献中出现的表述才算"标准术语"，列同义术语清单
   - B.3 按本 mode 原有标准执行（量表有原始出处 / 样本数据可申请 / 案例有一手材料 / 基线可复现）
   - B.4 V 空白：三组检索式（P×X / X×Y / P×Y）近 3 年命中均 ≤5 篇、且无同 PXYV 组合 → 空白成立；任一组命中超限 → 列出最接近的文献，进 B.5 对位看差别
3) 每个动作产出校准表一行：维度 | 检索式 | 命中数 + 代表文献 | 判定 | 依据

## B.5 近邻对位（novelty 的实证检验 —— 锚块 neighbors 字段在这里产出）

- 近邻论文定义：与本选题在 P（理论/范式）、X（对象）、Y（语境/领域）三维中至少共享两维的已发表论文
- 检索机制：① 三组检索式 P×X / X×Y / P×Y（近 5 年，各取被引最高的若干篇）② 对标论文（若提供）的参考文献与施引文献 ③ 合并去重，按共享维度数排序取前 5，每篇标注共享的是哪两维
- 用户动作（教练先给对位句草案）：每篇一句对位 ——"它做了 ___，我的差别是 ___"
  5 句都写不出差别 → 选题撞车，回 A.4 调整 X 或 Y
- 产出：近邻表（引文 | 共享维度 | 对位句）

输出 → `选题-B-calibration.md`（中间产物 · 断点续跑用；含校准表 + 近邻表）

## C.0 选题压力测试（先做一轮，再产出标志成果）

- 四个受力面，教练对每面给出「最强的一条质疑」+「一条回应草案」，用户逐面判断（接受质疑并修改 / 采纳回应）：
  1. P-X 适配：这个理论/范式最解释不了现象的哪一面？需要补充 / 更换视角吗？
  2. 新颖性：对照 B.5 近邻表，撞车风险最大的是哪一篇？对位句立得住吗？
  3. 可行性：数据 / 样本 / 方法链条上最薄弱的一环是什么？
  4. 价值：教练检索目标期刊近 5 年是否发过同一对话的论文（检索式限定该刊 + P×X / X×Y 关键词）——命中 ≥1 篇 → 对话在该刊存在，列出篇目作证据；命中 0 篇 → 给出检索记录，建议换刊或调整定位。用检索证据说话，不要求用户凭空判断
- 本 mode 侧重：mode ①③ 重点测 1、2；mode ②④ 重点测 2、3（其余两面也要过，但从简）
- 不打分、不设门槛；4 组质疑 + 回应作为「压力测试记录」写入 选题.md 附录

## 【贡献阶梯 · 写 选题.md 时 V 必须标注定级】

- L1 复制 / 语境扩展：同一 P 换一个 X / Y
- L2 机制 / 条件：新中介、新调节、新边界条件
- L3 构念 / 方法：新概念、新测量、新方法
- L4 假设挑战：推翻或改写文献中被想当然的命题

（依据：L1-L3 ≈ Colquitt & Zapata-Phelan 2007, AMJ 50(6) 理论贡献五级量表的 1-2 / 3-4 / 5 级；L4 ≈ Alvesson & Sandberg 2011, AMR 36(2) problematization + Corley & Gioia 2011, AMR 36(1) revelatory。量表源自管理学：mode ①② 直接适用，mode ③ 类比适用，mode ④ 的方法贡献按"增量改进 → 已有方法新应用 → 新方法 → 新范式"类推）

教练给当前 V 定级，并问一次："往上推一级长什么样？代价是什么？" 推不推用户定。
