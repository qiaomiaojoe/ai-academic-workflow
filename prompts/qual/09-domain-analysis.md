# 09 · Domain Analysis (Spradley 文化领域分析)

> **Neuman 出处**：Ch 14, "Domain and Scheme Analysis" · p.490-492
>
> **何时用**：你有人类学/田野/民族志类数据, 想找出研究对象**自己**的文化分类系统
>
> **前置**：田野笔记 / 访谈 / 观察记录已收集

---

## 📖 Neuman 怎么讲

Spradley (1979) 的 domain analysis 来自认知人类学:
> The basic unit in a cultural setting is a **cultural domain**, an organizing idea or concept.

**三种 domain**:
1. **Folk domain**: 用研究对象自己的语言 (argot)——你只听他们怎么说
2. **Mixed domain**: 用对象的语言但加你的概念
3. **Analytic domain**: 用研究者+理论的概念——当对象自己**没意识到**这个分类时

**Domain 三部分**:
- **Cover term**: 域的总称 (如"witness")
- **Included terms**: 子类 (如"defense witness", "expert witness")
- **Semantic relationship**: 怎么联结 (如"is a kind of")

**九种 semantic relationship** (Chart 1, p.491):
| 关系 | 例 |
|------|---|
| Is a type of | bus is a type of motor vehicle |
| Is a part of | tire is a part of car |
| Is a way to | cheating is a way to get high grades |
| Is used for | train is used for transporting goods |
| Is a reason for | high unemployment is a reason for public unrest |
| Is a stage of | the charge is a stage of a battle |
| Is a result of | acid rain is a result of coal power |
| Is a place for | town square is a place for mob |
| Is a characteristic of | spiked hair is a characteristic of punks |

**六步 (Expansion Box 5)**:
1. 反复读田野笔记
2. 把细节心理打包成几十个组织性 ideas
3. 从笔记里发掘新 ideas (基于主观意义 / 组织性 idea)
4. 找 ideas 之间的关系
5. 通过比较组织成更大的群组
6. 重组并连接成更广的整合 themes

---

## ⚠️ 必须你自己判断的部分

- **是用 folk / mixed / analytic domain**: 分析深度不同
- **cover term 怎么挑**: 必须真有所辖, 不能空中飞
- **semantic relationship 选哪种**: 同样数据用"is a kind of" vs "is used for" 出来不同 domain

## ✂️ Prompt 模板

```
我要按 Spradley / Neuman (2014) Ch 14 的 domain analysis 分析数据。

【数据类型】[ ] 访谈逐字稿 [ ] 田野笔记 [ ] 公开文档 [ ] 多源混合
【数据】[贴]

【要做哪类 domain】
[ ] Folk domain — 完全用对象语言
[ ] Mixed domain — 对象语言 + 我的少量概念
[ ] Analytic domain — 我的理论概念 (对象未必自觉)

【关注的研究主题】[一句话, 如 "海外华人怎么分类'家'的概念"]

【请按 Spradley 6 步做】

**Step 1**: 反复扫数据, 列出**对象用过的关键词**
- 不要总结, 只列原词
- 标 ID

**Step 2**: 心理打包成 organizing ideas
- 试图把关键词聚成 30-50 个粗类
- 给每个粗类一个临时名

**Step 3**: 找 cover term + included terms
- 对每个粗类, 找 cover term (总称) + included terms (子类)
- 用什么 semantic relationship?
  · "is a kind of" / "is a way to" / "is a result of" / "is a part of" / etc (见 Chart 1)
- **每个 cover term 至少 3 个 included terms 才有意义**

**Step 4**: 写 domain analysis worksheet (按 Neuman Example Box 1 格式)
| Included Terms | Semantic Relationship | Cover Term |
|----------------|----------------------|------------|
| [list] | is a kind of | [cover] |

**Step 5**: 跨 domain 比较
- 哪些 domains 在结构上类似? (相同 semantic relationship)
- 哪些 domains 互相矛盾? (一个对象在两个 domain 里被分类不同)

**Step 6**: 整合到更广的 themes
- 多个 domains 是否指向某个**更深的文化分类逻辑**?
- 给一个理论性总结

【硬约束】
- folk/mixed domain 必须用对象的原词, 不能用研究者发明的词
- 至少 3 个 included terms 才能算一个 domain
- semantic relationship 必须从 9 种里选 (或你明确说"自创的关系"+ 理由)
- 不许虚构 included terms

【输出】
- 每个 domain 一份 worksheet
- 末尾整合 themes
- "对象语言"和"研究者语言"分开标记 (避免混入研究者投射)
```

## 📌 演示填法 (假设访谈数据)

```
【数据】12 段海外华人对"家"的访谈
【domain 类型】mixed (对象语言 + 我的少量概念)

期望产出 1 个 domain:

Cover term: "家" (中文原词)
Semantic relationship: "is a kind of"
Included terms (来自访谈原话):
- "我爸妈那个家"
- "我自己买的房子"  
- "中国"
- "唐人街"
- "我和老公的小窝"
- "我心里的家"

发现: "家" 在海外华人话语里**至少 6 种**, 横跨物理-亲属-民族-国族-情感
跨 domain 比较: "家" vs "故乡" — 后者更窄 (只有民族/国族层) 
                "家" vs "归属"— 后者更抽象
整合 theme: "家" 是 multi-scalar concept, 不能简单翻译为 home
```
